Rust中的引用与借用（References and Borrowing）

在 Rust 中，借用（Borrowing） 允许你在不转移所有权的情况下使用数据。这一节详细讨论了如何通过引用（References）来借用数据以及相关的规则，确保内存安全和避免数据竞争。

1. 什么是引用（References）？

引用是指向数据的指针。Rust 提供了两种类型的引用：

  • 不可变引用（Immutable references）：允许你读取数据，但不能修改它。
  • 可变引用（Mutable references）：允许你修改数据。

引用使得多个函数或变量可以共享数据，而不需要拥有数据的所有权。

2. 引用的基本规则

（1）不可变引用

  • 不可变引用允许你读取数据，但不允许修改数据。
  • 多个不可变引用：你可以在同一时间拥有多个不可变引用。

不可变引用的语法：

let s = String::from("hello");
let r1 = &s;  // 不可变借用
let r2 = &s;  // 另一个不可变借用
println!("{}", r1);  // 使用不可变引用
println!("{}", r2);  // 使用另一个不可变引用
  • 你可以同时拥有多个不可变引用（r1 和 r2）来读取同一个数据。

（2）可变引用

  • 可变引用允许你修改数据，但在同一时间只能拥有一个可变引用。
  • Rust 的设计保证了可变引用期间，数据不会同时被其他代码访问，从而避免了数据竞争的问题。

可变引用的语法：

let mut s = String::from("hello");
let r1 = &mut s;  // 可变借用
r1.push_str(", world");  // 修改数据
println!("{}", r1);  // 输出修改后的数据
  • 在上述代码中，r1 是 s 的可变引用，可以修改 s 的内容。

（3）不可变引用与可变引用不能共存

  • 不可变引用和可变引用不能同时存在，即一个数据要么只能被不可变借用，要么只能被可变借用。
  • 如果你已经有一个可变引用，不能同时创建不可变引用，反之亦然。

示例：

let mut s = String::from("hello");
let r1 = &mut s;  // 可变借用
// let r2 = &s;    // 错误：不能同时拥有不可变和可变引用
println!("{}", r1);
  • 上面的代码会出错，因为在 r1 是可变引用的情况下，不能同时存在不可变引用。

3. 借用的作用与规则

我们将创建一个引用的行为称为 借用（borrowing）。正如现实生活中，如果一个人拥有某样东西，你可以从他那里借来。当你使用完后，必须还回去。因为我们并不拥有它的所有权。

借用与所有权的关系

  • 借用（Borrowing）允许你借用数据而不转移所有权。这意味着原始的所有者仍然拥有数据的所有权，而借用者只能使用该数据。
  • 通过借用，Rust 能够让多个部分共享数据而不需要复制数据，从而提高程序的性能。

借用的安全性

  • Rust 的借用规则确保了即使在并发场景下，数据也不会被多次修改。Rust 编译器通过所有权和借用检查在编译时保证内存的安全。

借用的生命周期

  • 借用也与生命周期（Lifetime）紧密相关，确保借用数据不会超出原始数据的生命周期。Rust 会在编译时检查生命周期，避免了悬空引用。

4. 创建引用时的细节

  • 引用的生命周期：引用的生命周期与原数据的生命周期是相互关联的。Rust 会确保引用不会在原数据被销毁后仍然存在，从而避免了悬空引用（dangling references）。
  • 切片类型：引用本质上是一个指向内存的指针，Rust 中的切片（&str 或 &[T]）是引用类型，用于表示对一部分数据的引用。

5. 举例：

不可变引用

fn main() {
    let s = String::from("hello");
    let r1 = &s;  // 不可变引用
    let r2 = &s;  // 另一个不可变引用
    println!("{}", r1);  // 使用 r1 输出
    println!("{}", r2);  // 使用 r2 输出
}
  • 你可以拥有多个不可变引用来读取数据，但不能修改数据。

可变引用

fn main() {
    let mut s = String::from("hello");
    let r1 = &mut s;  // 可变引用
    r1.push_str(", world");  // 修改数据
    println!("{}", r1);  // 输出修改后的数据
}
  • 只有一个可变引用可以存在，且它可以修改数据。

错误示例：同时拥有不可变和可变引用

fn main() {
    let mut s = String::from("hello");
    let r1 = &mut s;  // 可变引用
    // let r2 = &s;    // 错误：不能同时拥有不可变和可变引用
    println!("{}", r1);
}
  • 在一个可变引用存在的情况下，无法同时拥有不可变引用。

    //rintln!("String is {}, len is {}",str1, len); //error owener_ship(str) this parameter takes ownership of the value
    println!("String is {}, len is {}",str2, len);


    let str3 = String::from("refrence");
    //如何处理呢？ 使用引用
    let refrence_len = calculate_length_refrence(&str3); //& 符号就是 引用，它们允许你使用值但不获取其所有权，只能使用，无法修改
    println!("String is: {}, len is {}",str3, refrence_len);

    //如果需要在calculate_length_refrence中修改str4怎么办？ 使用可变引用
    let mut str4 = String::from("mut_refrence");
    let mut_refrence_len = calculate_length_mut_refrence(&mut str4); //&mut 符号就是创建可变引用
    println!("String is: {}, len is {}",str4, mut_refrence_len);

}

fn scope(){
    {                      // s 在这里无效, 它尚未声明
        let s = "hello";   // 从此处起，s 开始有效  s（&str）：字符串切片，它是指向栈外存储（常量区）数据的引用。它存储的只是一个指针，指向 "hello" 这个常量数据，栈上分配。
        let s1 = String::from("hello"); // String 类型的数据，它在堆上分配内存来存储字符串数据（"hello"），并在栈上存储指向堆内存的指针，堆上分配。
        println!("s=s1? :{}",s==s1); //s == s1 会将 String 类型的 s1 自动转换为 &str，然后比较 s 和 s1 的内容
    }                      // 此作用域已结束，s和s1 不再有效
}

fn owner_ship_string(str:String){
    println!("{}",str); 
}//作用域结束 str会被drop

fn owner_ship_integer(x:i8){
    println!("{}",x); 
}

fn calculate_length(s: String) -> (String, usize) {
    let length = s.len(); // len() 返回字符串的长度
    (s, length) //将s的所有权返回
}

fn calculate_length_refrence(s: &String) ->  usize {
    //s.push_str(", world"); //尝试修改会报错 cannot borrow `*some_string` as mutable, as it is behind a `&` reference
    return s.len();
}

fn calculate_length_mut_refrence(s: &mut String) -> usize{
    s.push_str(", calculate_length_mut_refrence");
    return s.len();
}

6. 总结：

Rust 的借用机制通过引用和所有权系统有效地防止了数据竞争和内存安全问题。引用允许共享数据，但根据 Rust 的规则：

  • 你可以通过不可变引用安全地读取数据。
  • 你可以通过可变引用修改数据，但不能同时拥有多个可变引用。
  • 引用的生命周期与原数据的生命周期是紧密绑定的，Rust 在编译时确保了内存安全。

通过这些机制，Rust 保证了内存的安全和并发访问的安全性，同时避免了垃圾回收的开销。
