Rust 方法语法(Method Syntax)

Awsome
Rust
发布日期:   2025-02-21
更新日期:   2025-02-21
文章字数:   1.6k
阅读时长:   7 分

Rust 提供了一种 方法（method） 语法，让我们可以将行为与结构体（struct）、枚举（enum）或 trait 关联起来。这一节介绍了 如何定义和调用方法、self 的作用、方法 vs 关联函数的区别 等内容。

1. 方法的定义

方法与普通函数类似，但它们必须在 impl 代码块中定义，并且第一个参数通常是 self，代表该方法作用于的实例：

struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {  // `self` 代表当前结构体的实例
        self.width * self.height
    }
}

fn main() {
    let rect = Rectangle {
        width: 30,
        height: 50,
    };

    println!("The area of the rectangle is {} square pixels.", rect.area());
}

✅ rect.area() 直接调用方法，而不是 area(&rect)，更符合直觉。

2. self 参数解析

self 参数在 impl 代码块的方法中，表示该方法作用于的实例：

self 参数类型 说明 适用场景
&self 不可变借用 只读访问字段，不修改结构体
&mut self 可变借用 修改结构体字段
self 获取所有权 方法调用后，实例被移动

1. &self（不可变借用）

impl Rectangle {
    fn area(&self) -> u32 {  
        self.width * self.height
    }
}
  • 只读取结构体字段，不会修改它。
  • self 等价于 &self: Rectangle。

✅ 适用于只访问数据，而不修改数据的方法。

2. &mut self（可变借用）

如果方法需要修改结构体字段，必须使用 &mut self：

struct Counter {
    value: u32,
}

impl Counter {
    fn increment(&mut self) {
        self.value += 1;
    }
}

fn main() {
    let mut counter = Counter { value: 0 };  // 实例必须是可变的
    counter.increment();  // 修改字段
    println!("Counter value: {}", counter.value);
}

✅ &mut self 允许修改 value，但调用方法的实例 counter 必须是 mut，否则会报错。

3. self（获取所有权）

有时候，方法需要 消耗（consume）结构体，即方法调用后该实例不再可用：

struct Person {
    name: String,
}

impl Person {
    fn into_name(self) -> String {  // 获取所有权
        self.name
    }
}

fn main() {
    let p = Person { name: String::from("Alice") };
    let name = p.into_name();  // `p` 在这里被移动，之后不能再使用
    println!("Name: {}", name);
}

✅ self 代表整个实例所有权的转移，p 在 into_name() 调用后不再有效。

3. 关联函数（Associated Functions）

方法不一定需要 self 作为参数，例如 new() 用于创建实例：

impl Rectangle {
    fn new(width: u32, height: u32) -> Self {  // Self 代表结构体类型
        Self { width, height }
    }
}

fn main() {
    let rect = Rectangle::new(30, 50);  // 直接使用结构体名调用
    println!("Rectangle: {} x {}", rect.width, rect.height);
}

✅ 关联函数 通常作为构造函数，用来返回 Self 类型的实例。

4. 方法调用的自动引用与解引用

在 Rust 中，方法调用时可以自动添加 & 或 &mut，也可以自动解引用 *，让代码更加简洁直观。

1 .自动引用（Auto Referencing）

Rust 允许在拥有所有权的对象上直接调用方法，而不需要手动添加 &。

struct Circle {
    radius: f64,
}

impl Circle {
    fn area(&self) -> f64 {
        3.14 * self.radius * self.radius
    }
}

fn main() {
    let circle = Circle { radius: 5.0 };

    println!("Area: {}", circle.area()); // 自动添加 &circle
}

✅ circle.area() 自动传递 &circle，所以 area(&self) 方法可以正常调用。

如果没有自动引用特性，每次调用都需要手动添加 &：

println!("Area: {}", (&circle).area());  // 需要手动添加 `&`

2 .自动解引用（Auto Dereferencing）

当方法接收 &self 时，Rust 会自动解引用 Box<T>、Rc<T>、Arc<T> 等智能指针，让方法调用更简洁。

use std::rc::Rc;

struct Foo {
    value: i32,
}

impl Foo {
    fn get_value(&self) -> i32 {
        self.value
    }
}

fn main() {
    let foo = Box::new(Foo { value: 42 });
    let shared_foo = Rc::new(Foo { value: 100 });

    println!("{}", foo.get_value());       // ✅ 自动解引用: (*foo).get_value()
    println!("{}", shared_foo.get_value()); // ✅ 自动解引用: (*shared_foo).get_value()
}

Rust 自动将 foo.get_value() 解析为 (*foo).get_value()，避免手动解引用。

3 .自动引用 & 解引用的规则

  • 如果方法需要 &self，Rust 自动添加 &。
  • 如果方法需要 &mut self，Rust 自动添加 &mut。
  • 如果 self 是 Box<T>、Rc<T>、Arc<T>，Rust 自动解引用。
struct Example {
    value: i32,
}

impl Example {
    fn print_value(&self) {
        println!("{}", self.value);
    }

    fn modify_value(&mut self) {
        self.value += 1;
    }

    fn consume(self) {
        println!("Consumed: {}", self.value);
    }
}

fn main() {
    let mut ex = Example { value: 10 };

    ex.print_value();   // ✅ 自动加 &ex
    ex.modify_value();  // ✅ 自动加 &mut ex
    ex.consume();       // ✅ 直接传递所有权
}

4 .结论

特性 作用 示例
自动引用 Rust 自动为方法调用添加 & 或 &mut circle.area() → (&circle).area()
自动解引用 Rust 自动解引用智能指针 foo.get_value() → (*foo).get_value()

Rust 的 自动引用 & 自动解引用 让方法调用更简洁，避免手动 & 和 *，提高代码可读性和开发效率。

5. 多个 impl 代码块

一个结构体可以有多个 impl 代码块：

struct Point {
    x: i32,
    y: i32,
}

impl Point {
    fn distance_from_origin(&self) -> f64 {
        ((self.x.pow(2) + self.y.pow(2)) as f64).sqrt()
    }
}

impl Point {
    fn new(x: i32, y: i32) -> Self {
        Self { x, y }
    }
}

fn main() {
    let p = Point::new(3, 4);
    println!("Distance: {}", p.distance_from_origin());
}

✅ Rust 允许多个 impl，使代码更清晰，避免 impl 代码块过长。

6. 方法 vs 关联函数

特性 方法（Method） 关联函数（Associated Function）
定义方式 需要 self 作为参数 没有 self
调用方式 instance.method() Type::function()
作用 作用于实例 作用于类型本身（如构造实例）

示例对比

struct User {
    name: String,
}

impl User {
    fn new(name: String) -> Self {  // 关联函数
        Self { name }
    }

    fn get_name(&self) -> &str {  // 方法
        &self.name
    }
}

fn main() {
    let u = User::new(String::from("Bob"));  // 关联函数调用
    println!("{}", u.get_name());  // 方法调用
}

7. 代码总结

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn new(width: u32, height: u32) -> Self {  // 关联函数（构造）
        Self { width, height }
    }

    fn area(&self) -> u32 {  // 方法（不可变借用）
        self.width * self.height
    }

    fn resize(&mut self, width: u32, height: u32) {  // 方法（可变借用）
        self.width = width;
        self.height = height;
    }

    fn into_tuple(self) -> (u32, u32) {  // 方法（获取所有权）
        (self.width, self.height)
    }
}

fn main() {
    let mut rect = Rectangle::new(30, 50);
    println!("Area: {}", rect.area());

    rect.resize(40, 60);
    println!("Resized Rectangle: {:?}", rect);

    let dimensions = rect.into_tuple();  // `rect` 被消费
    println!("Dimensions: {:?}", dimensions);
}

8. 总结

重点 说明
方法定义 方法必须定义在 impl 代码块中
&self 不可变借用，用于 只读访问
&mut self 可变借用，用于 修改实例
self 获取所有权，调用后实例被消费
关联函数 没有 self，通常用作 构造函数
自动引用 Rust 自动添加 &self 或 &mut self
多个 impl 允许多个 impl 代码块，使代码更清晰

Rust 的方法语法提供了清晰的所有权管理、可读性强的 API 设计，在实际开发中非常重要。🚀
